### commit规范
「type」message
示例:「refactor」作业批量修改为任务,重构项目目录结构
示例:「feat」XXX
```
feat：新功能（feature）
fix：修补bug
hotfix：紧急修复bug
test：增加测试
docs：文档（documentation）
style： 格式（不影响代码运行的变动）
refactor：重构（即不是新增功能，也不是修改bug的代码变动）
chore：构建过程或辅助工具的变动
